Feedback page

Technical Description

The feedback.html page features a form with various input types, including text fields, radio buttons, checkboxes, and a file upload. CSS is used to style the form, with flexbox for layout alignment and box-shadow for a modern look. The form includes validation for required fields, and the :required pseudo-class is used to highlight mandatory inputs. A star rating system is implemented using Unicode stars and CSS transitions for interactivity.

Accessibility

Accessibility is ensured by using <label> tags for all form inputs, making them screen-reader friendly. Required fields are marked with an asterisk and a visual cue. The star rating system is keyboard-navigable, and error messages are provided for invalid inputs. The form is designed to be accessible to users with disabilities, including those using assistive technologies.

Team page

Technical Description

The team.html page uses a responsive layout with Flexbox to display team members in a grid-like structure. Each team member is represented by a .team-member div, which contains an image and hidden member information that appears on hover using the :hover pseudo-class. The transition property is used to create a smooth scaling effect when hovering over team members. The page also includes a fixed background video for visual appeal, implemented using the <video> tag with autoplay, loop, and muted attributes.

Accessibility

Accessibility is addressed by providing alt text for images, ensuring screen readers can describe the team members' photos. The hover effect for member information is purely visual and does not interfere with keyboard navigation. The fixed video is muted to avoid disrupting users with screen readers or those sensitive to sound.

Content Page

Technical Description

The content_ST3.html page is structured with semantic HTML tags like <header>, <main>, and <section> for clear content organization. Images are centered using text-align: center and styled with fixed dimensions for consistency. The page uses simple CSS for typography and spacing, with a focus on readability. Each section is visually separated, and images are accompanied by descriptive alt text.

Challenges and Lessons Learned

Creating consistent styling and responsive designs across pages was challenging. Using a shared CSS file and testing on multiple devices helped resolve these issues. Integrating the sitemap navigation also required careful planning. These challenges highlighted the importance of thorough testing and planning for a seamless user experience.

Compliance

The pages comply with Janet regulations by using valid HTML5 and CSS3, ensuring accessibility with alt text and semantic tags. Deprecated tags were avoided, and external resources were optimized for performance, aligning with web standards and best practices.
