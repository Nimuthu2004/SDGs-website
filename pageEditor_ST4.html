Profile page

Technical Description

The profile.html page uses a three-panel layout (left-panel, middle-panel, right-panel) created with flexbox (display: flex) to organize user information, skills, and contact forms. CSS properties like box-shadow, border-radius, and padding are used to create a visually appealing design. The page includes a video background with position: fixed and z-index: -1 to ensure it doesn’t interfere with content readability.

Accessibility

Accessibility is addressed by providing a clear structure with semantic HTML tags like <header> and <main>. The video background is muted and looped, ensuring it doesn’t distract users with cognitive disabilities. The form inputs include required attributes and placeholders to guide users. The skill bars use contrasting colors (background-color: #4CAF50) to make progress visually clear, and the table for availability uses clear labels and symbols (✔ and ✖) for easy interpretation.

Sitemap page

Technical Description

The sitemap.html page uses a grid-like layout with buttons to navigate through different sections of the website. The layout is created using flexbox (display: flex) for responsive alignment of buttons and columns. CSS properties like gap, padding, and border-radius are used to create a clean and user-friendly interface. The page also includes a background image with background-size: cover to ensure it scales properly across devices.

Accessibility

Accessibility is improved by using descriptive button labels (onclick="location.href='splash.html'") and clear navigation links. The background image is purely decorative, so it doesn’t interfere with screen readers. The use of contrasting colors for buttons (background: rgb(56, 83, 222)) and text (color: white) ensures readability for users with visual impairments. The page is also keyboard-navigable, allowing users to tab through buttons.

Content Page

Technical Description

The content_ST4.html page uses a structured layout with HTML sections and CSS styling to present information about ocean conservation. Key HTML tags like <section> and <ul> are used to organize content, while CSS properties such as background-color, border-radius, and padding enhance the visual appeal. The page also includes images with responsive sizing using max-width: 100% and height: auto to ensure they adapt to different screen sizes. The use of semantic tags like <header> and <h2> improves readability and SEO.

Accessibility

Accessibility is addressed through the use of semantic HTML tags, which help screen readers interpret the content. Alt attributes are provided for images (alt="Ocean Cleanup Project") to describe visual content for visually impaired users. The color contrast between text and background is carefully chosen to ensure readability, and the responsive design ensures the page is accessible on various devices.

Challenges and Lessons Learned

A challenge was integrating a video background without compromising content readability. This taught the importance of using z-index and position: fixed to keep the video in the background. Another lesson was designing a responsive three-panel layout that adapts to different screen sizes.

Compliance

The page complies with accessibility standards by using semantic HTML tags and providing clear labels for form inputs. The video background is muted and looped, ensuring it doesn’t interfere with accessibility. The skill bars and availability table use clear visual indicators, and the color contrast meets WCAG guidelines.
