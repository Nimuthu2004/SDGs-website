Home page

Technical Description

The home.html page is structured with a <header>, <main>, and <footer>, using semantic HTML for clarity. The <main> section features a mission statement and a grid of cards created with Flexbox, each linking to different content pages. CSS is used to style the cards with hover effects, shadows, and transitions, enhancing interactivity. Key tags include <header>, <section>, and <div>, while key CSS properties like display: flex; and box-shadow ensure a responsive and visually appealing layout.

Accessibility

Accessibility is addressed through semantic HTML, alt attributes for images, and a muted video background to avoid disrupting screen readers. The page also uses proper heading hierarchy (<h1>, <h2>) for better navigation. However, visible focus states for keyboard users could further improve accessibility.

Table page

Technical Description

The table.html page uses a <table> to display marine conservation data, with rows and columns styled using CSS. The .styled-table class adds rounded corners, shadows, and alternating row colors for readability. Hover effects on table rows and images enhance interactivity. Key tags include <table>, <th>, and <td>, while key CSS properties like border-collapse and transition ensure a polished design.

Accessibility

Accessibility is addressed through proper table structure with <th> for headers and <td> for data, alt text for images, and descriptive links. However, visible focus states for keyboard users and ARIA labels for interactive elements could further improve accessibility.

Content Page

Technical Description

The content_ST2.html page organizes content into sections using <div> elements with the class .section. Each section contains a heading, image, and paragraph, styled with CSS for a clean, modern look. The .container class centers the content and adds a semi-transparent background with rounded corners and shadows. Images have a hover effect that scales them slightly, adding interactivity. Key tags include <h1>, <h2>, and <p>, while key CSS properties like border-radius and box-shadow enhance the design.

Accessibility

Accessibility is improved through proper contrast between text and background, alt text for images, and a logical heading hierarchy. However, the page could benefit from visible focus states for interactive elements to better support keyboard navigation.

Challenges and Lessons Learned

Balancing design, performance, and accessibility was the main challenge. Issues like video background optimization (home.html), readability with semi-transparent backgrounds (content_ST2.html), and responsive tables (table.html) were resolved through testing and best practices. The lesson was that thorough testing and semantic HTML are key.

Compliance

All pages comply with Janet regulations using semantic HTML, accessibility features (e.g., alt text, proper contrast), and responsive design. Multimedia is non-disruptive, meeting accessibility standards.
